# マーチンさんとエリックさんの違い

---

<big>

1. マーチンさんは実装のパターン
    - 不変にすることでシンプルに
    - IDは不要
    - データベースへの永続化も不要
    - 堅牢性の高い実装を実現
1. エリックさんは設計のパターン
    - 同一性が必要かどうか吟味
    - 不要ならシンプルな設計になる
    - 不変かどうかは本質じゃない

</big>
